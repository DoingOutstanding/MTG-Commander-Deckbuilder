# MTG Commander Deckbuilder

Local Flask web app that helps you build a Commander deck around a chosen
commander. For each candidate card, it scores how strongly the card
interacts with what's already in the deck and ranks suggestions by that
score. You add cards one at a time, and the suggestion list re-ranks
after every addition.

There are two ways to run it:

- **Native desktop app (recommended for end-users)** — see
  [`electron/README.md`](electron/README.md). Builds a self-contained
  installer (Windows `.exe`, macOS `.dmg`, Linux `.AppImage`). No
  Python install, no terminal, no `.bat` files — just download and
  double-click.
- **Direct Python script (for development)** — see the quick-start
  below.

## Quick start (one click)

1. **Get Python.** If you don't have it: <https://www.python.org/downloads/>
   (3.10 or newer; on Windows make sure to check *"Add Python to PATH"*
   during install).

2. **Get the card data.** Download the Scryfall **Oracle Cards** JSON
   from <https://scryfall.com/docs/api/bulk-data> (~165 MB) and save the
   file inside this folder. The filename looks like
   `oracle-cards-YYYYMMDDHHMMSS.json`.

3. **Launch.**
   - **Windows** — double-click `run.bat`.
   - **macOS** — double-click `run.command`. (If macOS refuses the first
     time, right-click → Open and confirm once.)
   - **Linux** — `bash run.command` from a terminal, or just
     `python3 app.py` after running `python3 profile.py` once.

The launcher will install Flask if it isn't already, run `profile.py`
once to build `cards.jsonl` (about 30 seconds, only needed the first
time), start the server, and open your browser to
<http://127.0.0.1:5000>.

Close the launcher window to stop the server.

recreate `cards.jsonl` from `oracle-cards-*.json` on first run, so you
don't have to ship the generated file.

## Manual run (if the launcher doesn't suit you)

```bash
pip install flask
python profile.py        # builds cards.jsonl from oracle-cards-*.json
python app.py            # starts at http://127.0.0.1:5000
```

If the Scryfall JSON is somewhere other than this folder:

```bash
python profile.py "C:\path\to\oracle-cards-XXXX.json"
```

`profile.py` also looks in `~/Downloads` and `~/Desktop` automatically.

## XMage mode

The folder ships with `xmage_cards.txt` — a flat list of every card
[XMage](http://xmage.de/) currently implements (~31,000 names). When the
app starts it loads that list, and the **XMage** button in the top bar
filters the candidate pool to only cards XMage actually has. Useful for
making sure the deck you build is actually playable on XMage.

If you have a fresh checkout of the XMage source tree at
`../Mage.Sets/src/mage/cards/`, the app will use that instead — handy
if you want a more current list than the bundled one. To refresh the
bundled file from a fresh XMage checkout:

```bash
python -c "import os; root='../Mage.Sets/src/mage/cards'; \
names=sorted({fn[:-5].lower() for sub in os.listdir(root) \
for fn in os.listdir(os.path.join(root,sub)) if fn.endswith('.java')}); \
open('xmage_cards.txt','w').write('\\n'.join(names) + '\\n')"
```

If neither the bundled file nor the XMage source is present, the XMage
button is hidden silently and everything else works as normal.

### Personal exclusions: `xmage_excluded.txt`

If your installed XMage build is older than the source tree the bundled
list was built from, some recent cards (or cards whose mechanics aren't
fully implemented in the binary you're playing on) will appear in
suggestions even though XMage rejects them on deck import. Add them,
one per line, to `xmage_excluded.txt`:

```
# my XMage doesn't have the TMNT or Avatar sets yet
Knowledge Seeker
Ray Fillet, Man Ray
Technodrome
```

Names are normalized on read (case-insensitive, punctuation ignored), so
`Knowledge Seeker`, `knowledge seeker`, and `KnowledgeSeeker` all match
the same card. Comments start with `#`. The default file pre-excludes
the four cards reported above plus all 34 Ikoria mutate cards, since
those are commonly missing from older XMage binaries — delete those
sections (or the whole file) once your XMage gets updated.

## Auto-build

The **Auto-build** button (top bar) builds a 100-card deck around the
chosen commander in five phases. Spells come first, then the mana base
is built around the actual colored-pip distribution of those spells.

1. **Staples.** Add a short list of CI-legal mana rocks (Sol Ring, Mind
   Stone, Thought Vessel — plus Arcane Signet, Fellwar Stone,
   Commander's Sphere in multi-color) and "any-CI-color" utility lands
   (Reliquary Tower, Rogue's Passage — plus Command Tower, Exotic
   Orchard, Reflecting Pool in multi-color, Path of Ancestry in 3C+).
   Cycle lands (originals/shocks/triomes/etc.) are NOT picked here —
   they're deferred to Phase 4 so they can be matched to actual pip
   demand.
2. **Tribal core.** If the commander references a tribe AND is itself
   of that tribe (Sliver Hivelord, Krenko Mob Boss, Edgar Markov,
   Marrow-Gnawer, etc.), add 25 popular members of that tribe before
   the synergy ranker runs. This solves the chicken-and-egg problem
   where the ranker is dominated by mana-rock / land mechanic_overlap
   edges from the staples and won't favour tribe members until many
   are already in the deck.
3. **Synergy fill for spell slots.** Greedy-pick non-land suggestions
   until the deck has 63 spells (= 100 − 1 commander − 36 lands).
4. **Pip-aware mana base.** Count the colored mana pips of every
   non-land card in the deck (commander + spells), then:
   - Order CI pairs/triples by combined pip weight; pick cycle members
     (4 originals + 4 battlebonds + 3 triomes + 4 shocks + 3 checks +
     2 slow/filter/surveil/bounce) starting with the heaviest pairs.
     A White-heavy 5C sliver deck gets Plateau, Tundra, Sacred
     Foundry, Hallowed Fountain, Godless Shrine, Temple Garden,
     Savannah, Bountiful Promenade, etc. — all the W-pairs come first.
   - Phase 3a-equivalent: fill the remaining nonbasic-land slots by
     color-fix score (Mana Confluence, Forbidden Orchard, Prismatic
     Vista, Fabled Passage).
   - Distribute basics across CI colors using the largest-remainder
     (Hamilton) method on pip shares — a deck with W=22, U=15, B=13,
     R=18, G=17 pips and a 12-basic budget gets 3 Plains, 2 Islands,
     2 Swamps, 3 Mountains, 2 Forests.
5. **Top-up.** Rare edge case where the synergy ranker exhausted before
   the spell target — fall through to fill remaining slots.

Ash Barrens and other "any color via cycling" lands are NOT credited as
fixing — their cycle ability removes the land from your battlefield,
so they score as colorless utility (≈0.5).

### Persistent pair-score cache

Pair scores are deterministic given the card profiles, so the app
memoises every `score_pair(a, b)` result in `pair_cache.pkl.gz` next to
`app.py`. The cache loads on startup, fills as `auto_build` runs, and
saves automatically (after each auto-build and on clean shutdown via
`atexit`). It also drives a neighbor index — the ranker iterates only
candidates that have a non-zero edge with at least one card in the
current deck, dropping the candidate scan from ~30k cards to ~7k for a
fully-built deck.

To pre-warm the cache for distribution, run

```bash
python prewarm_cache.py
```

on a dev machine. This auto-builds 20 popular commanders and saves the
resulting cache (~6–10 MB compressed). Ship that file alongside the
app and end-users get instant ranking for those commanders out of the
box; the cache continues to grow organically as users explore.

Mana-quality multipliers used by the color-fix score (descending):

|cycle|multiplier|notes|
|---|---|---|
|Original duals (Tundra)|1.00|no drawback|
|Battlebonds (Bountiful Promenade)|0.95|untapped in EDH|
|Check lands (Dragonskull Summit)|0.90|matches typical 2C usage|
|Shock lands (Sacred Foundry)|0.85|one-time 2 life|
|Pain lands (Mana Confluence)|0.85|per-tap 1 life|
|Filter lands (Cascade Bluffs)|0.85|filter cost is mild|
|Slow lands (Dreamroot Cascade)|0.80|untapped after 2 lands|
|Surveil lands (Elegant Parlor)|0.78|tapped + scry|
|Bounce lands (Boros Garrison)|0.75|tapped, but +1 land drop|
|Triomes (Xander's Lounge)|0.70|always tapped, cycling|
|Vanilla guildgates|0.70|always tapped|

Conditional any-color lands (Ancient Ziggurat, Cavern of Souls, Plaza
of Harmony, Gond Gate, Glimmervoid, Rainbow Vale, Gemstone Mine,
Forbidden Orchard, Castle Sengir, Crypt of the Eternals, etc.) get
additional penalties for their specific drawbacks — type restrictions,
self-sacrifice triggers, finite charges, opponent-grants-control,
flip/transform-conditional production, activated mana costs, etc. —
so they can't crowd out true fixing.

Arena Alchemy `A-` rebalanced variants are dropped from the candidate
pool entirely (they're digital-only and not legal in paper Commander
or in XMage).

## How it works

Three layers:

1. **Profiler** (`profile.py`) — reads each card's oracle text and
   extracts ~30 structural fields: card types, subtypes, color identity,
   tribes referenced, mechanics produced, mechanics cared about, active
   zones, plus structural flags from the combo detector
   (`lifegain_p1p1_target`, `creates_copy_with_tap`,
   `wins_on_empty_library`, etc.). Output: `cards.jsonl`.

2. **Interaction scorer** (`interactions.py`) — for any pair of cards,
   computes a list of typed edges (tribal, mechanic_overlap,
   resource_flow, combo) with a weight and zone tags. Edges are
   symmetric — `score_pair(a, b) == score_pair(b, a)`.

3. **Flask app** (`app.py`) — UI plus the ranking loop. For every card
   in the candidate pool (Commander-legal, color-identity ≤ commander,
   not already in deck), sums `score_pair(candidate, deckmate)` over
   every deckmate. Top 36 are shown as suggestion cards with images.

Color identity, banlist, and singleton are enforced. Suggestions never
include cards that violate the format.

## What "interaction" means here

The scorer accumulates four families of edges:

- **Tribal.** Card A references a subtype T (in oracle text) and card B
  is of subtype T. Lord-style effects (both are tribe T and at least one
  references it) get an extra bonus. Edge weight is scaled by
  `1/sqrt(tribe_size/100)` so being a Goblin (~500 cards) is worth less
  per edge than being a Sliver (~100 cards).
- **Mechanic overlap.** Tags like `p1p1_counters`, `tokens_treasure`,
  `dies_trigger_payoff`, `landfall`, `lifegain`. A card "produces" a tag
  if its oracle text matches a producer pattern, "cares about" a tag if
  it matches a consumer pattern. Producer→carer is a stronger edge
  (`resource_flow`) than producer→producer overlap (`mechanic_overlap`).
- **Combo.** A small set of pre-coded patterns from the combo detector:
  Heliod+Ballista (`lifelink_p1p1`), Splinter Twin/Kiki+untapper
  (`kiki_twin`), Deadeye+Drake (`deadeye_drake`),
  Thoracle+Consultation (`thoracle`), Mikaeus+Triskelion
  (`mikaeus_trike`), Sanguine+Exquisite (`sanguine_exquisite`). Each
  combo edge is worth 100 points — far more than any synergy edge — so
  combo pairs always surface to the top.
- **Zone tags.** Each edge carries a list of `(zone_a, zone_b)` pairs
  — `BF/BF` for permanents-in-play interactions, `STACK/BF` for cast
  triggers, `GRAVE/BF` for graveyard payoffs, etc. The active-zone
  filter prunes nonsense edges (Lightning Bolt does nothing in the
  graveyard except when flashback-equivalent text is present, so it
  doesn't get `GRAVE/BF` edges).

## Examples that surface correctly

- **Krenko, Mob Boss** → Siege-Gang Commander, Mogg War Marshal, Goblin
  Instigator (every top suggestion is a Goblin token-maker).
- **Atraxa, Praetors' Voice** → Giada Font of Hope, Herald of War,
  Youthful Valkyrie (proliferate + counter-pumpers).
- **Yuriko, the Tiger's Shadow** → Ninja tribal cards (Silver-Fur
  Master, Ingenious Infiltrator, Higure, Walker of Secret Ways).
- **Heliod, Sun-Crowned** → Walking Ballista, Monoskelion, Triskelion
  at 112 points each (combo edge of 100 + counter resource flow).

## Known limitations

- **Tribal vs. specific synergy.** When a commander is a member of a
  large tribe (Wizards, Soldiers, Humans), tribal edges can dominate
  more specific mechanic synergies. For Niv-Mizzet, Parun, the Wizard
  tribal cards outscore Curiosity (the iconic 2-card combo partner)
  because there are more Wizards than Curiosity-equivalents. The
  `1/sqrt` scaling helps but doesn't fully balance. Tunable in
  `interactions.py` (`TRIBAL_EDGE_BASE`, `TRIBAL_LORD_BONUS`).
- **Static text only.** No game simulation — interaction is structural
  match, not actual rules-engine play. Most pairs the simulator would
  catch are caught here, but some idiosyncratic interactions
  (replacement effects, layered continuous effects) need the real
  engine.
- **No format-rules tuning per archetype.** All cards in the candidate
  pool are scored the same way. There's no awareness of mana curve,
  ramp/draw/removal balance, or category quotas — the suggestion list
  is pure synergy density. For a focused build that's fine; for a
  balanced "good-stuff" deck you'd want category constraints on top.
- **Single-user, in-memory state.** The deck is held in a process-wide
  dict. Restart the server and you lose the deck. Add JSON
  persistence in `app.py:STATE` if you want to keep state across
  sessions.
- **Combo detection is not exhaustive.** Six 2-card combo families are
  pre-coded. The combo detection report (`xmage_combos_scryfall.md` in
  the workspace folder) covers eight families across 360 pairs; only the
  most reliably detected six are used here.
